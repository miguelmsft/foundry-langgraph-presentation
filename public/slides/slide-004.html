Why Hosted Agents?

What You Own

  • Agent logic & orchestration
  • Framework choice (LangGraph, etc.)
  • Custom tools & integrations
  • Model selection & prompts
  • Business logic & state

What Foundry Manages

  • Container hosting & auto-scaling
  • Entra ID identity & authentication
  • Conversation orchestration
  • OpenTelemetry tracing & metrics
  • Publishing to Teams / M365 Copilot

Full code control + managed infrastructure. Use any framework. Use any library. Foundry handles the rest.

The value proposition is simple: you get the control of writing your own code — any framework, any library, any orchestration pattern — but you don't have to manage infrastructure. No Kubernetes clusters, no load balancers, no certificate management. Foundry handles scaling, identity (Entra ID), conversation management, observability, and even publishing. The Azure Architecture Center recommends Hosted Agents when you need: models not in the Foundry catalog, deterministic control over execution paths, custom tool integrations, code auditing for compliance, or advanced deployment practices.
