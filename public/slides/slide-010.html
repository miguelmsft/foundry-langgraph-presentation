Dockerfile — Packaging Your Agent

FROM python:3.12-slim WORKDIR /app COPY requirements.txt . RUN pip install --upgrade pip && \ pip install --no-cache-dir -r requirements.txt COPY . /app/LogisticsAgent/ # Bake the SQLite database at build time RUN python -c "import sys; sys.path.insert(0, '/app'); \ from LogisticsAgent.data.init_db import init_database; \ init_database()" EXPOSE 8088 ENV PYTHONPATH=/app CMD ["python", "-m", "LogisticsAgent.main"]

⚠️ Always build with --platform linux/amd64 (remote builds handle this automatically)

The Dockerfile is straightforward. Python 3.12-slim base, install dependencies, copy code, expose port 8088, and set the entrypoint. Two things worth noting: First, we run pip install --upgrade pip because the base image ships with an older pip that can fail on certain package extras. Second, we bake the SQLite database into the image at build time. The critical architecture requirement is Linux AMD64. Since we use remoteBuild: true, ACR handles this correctly.
