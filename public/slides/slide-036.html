For Hosted Agents:

  • MCP: Use langchain-mcp-adapters or Foundry native MCPTool
  • A2A: Can be A2A client (indirect) — not A2A server natively (yet)
  • Both: Complementary — MCP for tools, A2A for agent collaboration
Two protocols to be aware of. MCP — Model Context Protocol — connects agents to tool servers. It's GA in Foundry. A2A — Agent-to-Agent — is Google's open protocol for inter-agent communication, now under the Linux Foundation. It's in preview in Foundry via A2APreviewTool. For Hosted Agents, you can be an A2A client but the hosting adapter only exposes the Responses API. The key takeaway: MCP for tools, A2A for agents. They're complementary.
