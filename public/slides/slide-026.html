How the Agent Works

💬 User
“Analyze inventory and identify critical shortages”
🧠 LLM: “What tables exist?”
🔧 list_tables
→ [“inventory”, “shipments”]
🧠 LLM: “Schema next”
🔧 get_table_schema
→ {columns: [...], row_count: 25}
🧠 LLM writes SQL
→ WHERE qty < reorder_point
🔧 query_database (×5–8)
→ shortages, delays, values
📊 LLM writes final report
6 sections + emoji + recommendations
Here's the reasoning flow. The agent starts by discovering the tables, then learns the schema, then runs SQL queries — typically 5 to 8 of them — to gather data about shortages, delays, supplier performance, inventory values, and upcoming risks. Finally, it synthesizes everything into a structured report. The key point is that the agent decides autonomously what to do. You don't tell it which SQL to write or how many queries to run. Typically this takes about 15-20 seconds total.
