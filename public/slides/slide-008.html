This is the agent manifest — the identity card for your agent in Foundry. A few critical things to note. First: kind: hosted MUST be at the root level, NOT nested under a template: block. Older docs showed a nested format, but the current azd extension rejects it with a confusing error. Second: environment variables use ${VAR_NAME} syntax which gets resolved from Bicep outputs at deploy time. Third: the resources section tells azd what models to deploy — here we need gpt-5.4-mini.
