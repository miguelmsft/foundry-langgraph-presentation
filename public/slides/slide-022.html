Gotchas & Lessons Learned

#IssueWhat HappenedFix
1 agent.yaml schema kind: hosted nested under template: — azd rejected it Move to root level
2 Region East US 2 had capability host bugs Use North Central US
3 pip upgrade Base python:3.12-slim pip chokes on extras RUN pip install --upgrade pip
4 Soft-delete Redeploying with same name fails az cognitiveservices account purge
5 Model name gpt-5.4-mini vs gpt-54-mini Deployment name can't have dots
6 Starter template Must use azd-ai-starter-basic Don't hand-roll the Bicep
These are real issues we hit building this demo. Number 1 was the most frustrating — the agent.yaml schema changed, and older docs still show the nested format. Number 2: we initially deployed to East US 2 and the capability host creation failed silently. North Central US worked perfectly. Number 3 is a classic Python gotcha — always upgrade pip. Number 4: Cognitive Services accounts have a 48-hour soft-delete period. Number 5: Azure resource names can't have dots. And number 6: don't try to write your own Bicep — the starter template handles everything automatically.
