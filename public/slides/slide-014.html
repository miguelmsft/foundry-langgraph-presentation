The ReAct Pattern

Think → Act → Observe → Repeat

💬 User
“What parts are critically low on stock?”
🧠 THINKS
“I need to check what tables exist”
🔧 ACTS
→ calls list_tables tool
👀 OBSERVES
→ gets [“inventory”, “shipments”]
🧠 THINKS AGAIN
“Now I'll check the schema...”
... 5–8 iterations ...
📊 Final Report

This is what create_react_agent() builds — the dominant pattern for AI agents.

Reasoning + Acting = ReAct

ReAct — Reasoning + Acting — is the dominant pattern for AI agents. The LLM thinks about what to do, takes an action by calling a tool, observes the result, and then decides what to do next. This loop continues until the LLM decides it has enough information to answer. In LangGraph, create_react_agent() is a prebuilt function that constructs this exact graph. It creates two nodes — an agent node that calls the LLM and a tool node that executes tools — with a conditional edge that routes based on whether the LLM wants to use a tool or respond.
