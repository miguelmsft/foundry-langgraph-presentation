One Command Deployment: azd up

1. PROVISION (azd provision) ~5 min
Resource Group · Foundry Account + Project · Model Deployment (gpt-5.4-mini) · ACR · Application Insights · Capability Host ← auto-created!
2. BUILD (remote, no local Docker)
Sends Dockerfile to ACR · Builds linux/amd64 image
3. DEPLOY
Pushes image to ACR · Creates agent version · Starts container · Outputs: playground URL + endpoint URL

Setup: azd init -t Azure-Samples/azd-ai-starter-basic → azd ai agent init -m agent.yaml → azd up

Three setup commands and one deploy command — that's it. First, azd init pulls the official Foundry starter template which includes all the Bicep infrastructure. Then azd ai agent init imports your agent.yaml and wires it into azure.yaml. Then azd up does everything: provisions all Azure resources via Bicep, builds the container remotely, pushes to ACR, creates the hosted agent version, and starts it. The capability host gets auto-created by the Bicep template. After deployment you get the playground URL and the agent endpoint URL.
