azure.yaml — Deployment Configuration

requiredVersions: extensions: azure.ai.agents: >=0.1.0-preview name: foundry-langgraph-demo services: LogisticsAgentLG: project: src/LogisticsAgent host: azure.ai.agent # ← KEY: NOT host: containerapp language: docker docker: remoteBuild: true # ← No local Docker needed! config: container: resources: cpu: "1" memory: 2Gi deployments: - model: format: OpenAI name: gpt-5.4-mini version: "2026-03-17" name: gpt-54-mini sku: capacity: 10 name: GlobalStandard
The azure.yaml is what azd reads to understand how to deploy your agent. The most important line is host: azure.ai.agent — this tells azd this is a Foundry Hosted Agent, NOT a generic Container App. If you accidentally write host: containerapp, you'll deploy to Azure Container Apps directly and lose all the Foundry benefits. The remoteBuild: true setting is huge: your container gets built in Azure via ACR Build Tasks, so you don't need Docker installed locally at all.
