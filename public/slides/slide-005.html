Here's the architecture. Your code lives in a standard Docker container — Linux AMD64, listening on port 8088. Inside the container, the hosting adapter wraps your LangGraph agent into an HTTP server that speaks Foundry's Responses API. Foundry Agent Service sits around your container handling everything else: auto-scaling from 1 to 5 replicas, conversation state management, Entra ID authentication, and telemetry via Application Insights. The container specs range from 0.25 vCPU / 0.5 GiB up to 4 vCPU / 8 GiB.
