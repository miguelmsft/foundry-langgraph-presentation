The Hosting Adapter — One Line of Code

from azure.ai.agentserver.langgraph import from_langgraph adapter = from_langgraph(my_agent) adapter.run() # HTTP server on localhost:8088

What it handles:

  • 🔄 Protocol translation — Foundry Responses API ↔ LangGraph messages
  • 💬 Conversation management — Message serialization, state tracking
  • 📡 Streaming — Token-by-token streaming to clients
  • 📊 OpenTelemetry — TracerProvider, MeterProvider, LoggerProvider setup
  • 🔐 Authentication — Entra ID token flow integration
This is what makes the whole thing work. The azure-ai-agentserver-langgraph package — currently at version 1.0.0b17 in beta — provides a single function: from_langgraph(). You pass it your compiled LangGraph agent, and it wraps it into an HTTP server that speaks Foundry's Responses API. It handles all the complex protocol translation between Foundry's request/response format and LangGraph's native state messages. It also sets up OpenTelemetry automatically — TracerProvider, MeterProvider, LoggerProvider — with Azure Monitor exporters.
