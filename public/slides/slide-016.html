Giving Your Agent Tools

tools.py (from our demo)
import json, sqlite3 from langchain_core.tools import tool ALLOWED_TABLES = {"inventory", "shipments"} @tool def query_database(sql_query: str) -> str: """Execute a read-only SQL query against the logistics database. Args: sql_query: A SELECT SQL query. Only SELECT statements allowed. Returns a JSON string with columns, rows, and row count. Results truncated to 100 rows. """ normalized = sql_query.strip().upper() if not normalized.startswith("SELECT"): return json.dumps({"error": "Only SELECT queries are allowed."}) dangerous = {"INSERT", "UPDATE", "DELETE", "DROP", "ALTER", "CREATE"} if set(normalized.split()) & dangerous: return json.dumps({"error": "Query contains disallowed keywords."}) conn = sqlite3.connect("file:data/logistics.db?mode=ro", uri=True) cursor = conn.cursor() cursor.execute(sql_query) columns = [d[0] for d in cursor.description] rows = [list(r) for r in cursor.fetchall()[:100]] conn.close() return json.dumps({"columns": columns, "rows": rows})

🛡️ Safety: Read-only mode, keyword blocking, result truncation

Tools are just Python functions with the @tool decorator. Three things matter: First, the docstring — the LLM reads it to decide when and how to use the tool. Second, safety — query_database uses read-only mode plus dangerous keyword blocking. Results are truncated to 100 rows. Third, return format — tools should return strings, typically JSON, so the LLM can parse the result. We have three tools total: list_tables, get_table_schema, and query_database.
