Connecting to Foundry Models

from langchain_openai import AzureChatOpenAI from azure.identity import DefaultAzureCredential, get_bearer_token_provider # Auto-refreshing token provider — no API keys, no expiration issues token_provider = get_bearer_token_provider( DefaultAzureCredential(), "https://cognitiveservices.azure.com/.default", ) model = AzureChatOpenAI( azure_endpoint="https://my-foundry.openai.azure.com", api_version="2025-03-01-preview", azure_deployment="gpt-54-mini", azure_ad_token_provider=token_provider, # ← Auto-refreshes! )

🔑 Key point: get_bearer_token_provider auto-refreshes tokens.
Direct token fetching causes 401 errors after ~1 hour.

The model connection pattern is critical to get right. We use AzureChatOpenAI from langchain-openai. The crucial detail is using get_bearer_token_provider instead of fetching tokens directly. Token providers automatically refresh when tokens expire — typically after one hour. If you fetch a token directly and pass it as a string, your hosted agent will start failing with 401 errors after an hour. DefaultAzureCredential tries multiple auth methods in order: environment variables, managed identity, Azure CLI, and more.
