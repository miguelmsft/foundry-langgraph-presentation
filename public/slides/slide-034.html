Telemetry Verification

After sending 4 test prompts, we verified everything was tracked:

  • ✅ Foundry Responses API — 5 responses logged. All requests completed successfully with unique response IDs
  • ✅ Conversations API — 20 items tracked:
    • • 4 user messages (our prompts)
    • • 5 assistant responses (the reports)
    • • 11 function calls + outputs (tool invocations)
  • ✅ Trace IDs — Unique per request. Each prompt received a unique Trace ID for drill-down

Every tool call is captured — not just the final response.

We ran four different test prompts — stock shortages, inventory value, supplier performance, and active shipments — and then verified everything in Foundry's monitoring. Five responses were logged. Twenty conversation items were tracked — that includes the user messages, the assistant responses, AND every tool call the agent made. Each request got a unique Trace ID. How to inspect: go to ai.azure.com → Project → Tracing. The key insight: Foundry captures the full execution path, not just input and output.
