Deployment Flow

One-time setup:

# 1. Install the azd AI agent extension azd ext install azure.ai.agents # 2. Initialize from official starter template azd init -t Azure-Samples/azd-ai-starter-basic # 3. Import your agent definition azd ai agent init -m src/LogisticsAgent/agent.yaml

Deploy:

# 4. Provision + build + deploy azd up

Test & Monitor:

azd ai agent show # Check status azd ai agent invoke "Analyze..." azd ai agent monitor # Stream logs
The deployment workflow has two phases. Setup is three commands: install the AI agent extension, initialize from the official starter template that includes all the Bicep infrastructure, and import your agent.yaml which wires everything together. Then azd up is the one-command deployment: it provisions all infrastructure via Bicep, builds the container remotely in ACR, pushes the image, creates the hosted agent version, and starts it. After deployment, you have three lifecycle commands: show, invoke, and monitor. For cleanup, azd down tears everything down.
